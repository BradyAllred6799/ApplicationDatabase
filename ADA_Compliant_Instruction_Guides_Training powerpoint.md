## Slide 2 — What is WCAG 2.2 AA?

- **WCAG** = international accessibility standards from the W3C (World Wide Web Consortium)
- **4 principles (POUR):** Perceivable, Operable, Understandable, Robust
- **Level AA** = widely adopted, practical requirements that remove common barriers
- **In Word guides, AA compliance means:** clear Headings, descriptive links, alt text, accessible tables/TOC, and sufficient color contrast
- **Version 2.2 adds:** clearer focus indicators and alternatives to dragging interactions

## Slide 4 — Styles in Word: What? Why? How?

- **What:** Styles are built-in labels (Heading 1/2/3, Normal, Strong, Click) that define *function*, not just appearance.
- **Why:** Provide structure for navigation and Table of Contents generation, ensure consistency, and enable accessibility for screen readers.
- **How:** Select text, then apply a Style from **Home > Styles**. Avoid manual formatting.
- **Note:** Style tags do **not** transfer to PDF — this is a key reason we are moving away from PDFs.

| Style | Use |
|---|---|
| **Heading 1** | Guide title |
| **Heading 2** | Section headings |
| **Heading 3** | Subsections |
| **Normal** | Body text |
| **Strong** | Use instead of "bold" |
| **Click** | UI elements users select (buttons, menus, links) |
| **Bulleted List** | Use instead of the paragraph bullet tool |
| **Note** | Notes and warnings |

## Slide 6 — Lists and TOC (Make Navigation Work)

- Steps in a process should be **numbered**.
- Guidance outside a process uses **bullets** (standard circle; open circle for sub-bullets).
- An **automatic Table of Contents** is required for documents longer than 3 pages.
  - Insert via: **References > Table of Contents > Automatic Table 1 or 2**

## Slide 7 — What Every Document Needs for Compliance

### Structure

- Heading 1/2 hierarchy, numbered steps, bulleted guidance, auto TOC

### Text & Color

- Sufficient contrast; no color-only meaning (no highlights or font color changes without styles to convey importance)

### Links

- Descriptive link text; visually distinct (no underlines except for links)

### Images & Media

- Avoid images of text where possible
- Set all images in-line with text
- Add alt text or mark as decorative
- Include captions/transcripts if media is present

### Tables

- Header row, banded rows, first column enabled
- Repeat header row on each page
- No merged cells
- Include a caption
- Purpose-focused alt text

### Metadata & Framing

- **File > Info > Title** must match the Heading 1 title
- T&OD header and footer applied
- Spacing rules followed
- Set to read-only on publish

### Final Checks

- Run the **Accessibility Checker**
- Spot-check headings, links, tables, and TOC

## Slide 11 — DEMO: Links and the Click Style

- Use **descriptive link text** that describes what will happen when clicked.
- Apply the **Click style** to UI terms users interact with (e.g., Confirm, Continue, Submit).
- Three link types:
  - **Webpages** — link to an external URL
  - **Mailto** — link that opens a new email draft
  - **Headings within the document** — link to a section heading or bookmark

## Slide 14 — Publish-Ready Checklist

Before publishing, confirm all of the following:

- [ ] Heading hierarchy is correct (H1 → H2 → H3)
- [ ] Lists are used correctly (numbered for steps, bulleted for guidance)
- [ ] Table of Contents is inserted and updated (if 3+ pages)
- [ ] T&OD Header and Footer are applied
- [ ] Spacing rules are followed
- [ ] Metadata Title (File > Info) matches Heading 1
- [ ] Document is set to open read-only
- [ ] Tables are accessible (header row, no merged cells, alt text, caption)
- [ ] Links use descriptive text
- [ ] Images are in-line with text
- [ ] Alt text is added to all meaningful images
- [ ] Accessibility Checker has been run and issues resolved
