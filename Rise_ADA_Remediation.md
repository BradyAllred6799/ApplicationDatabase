# Rise Course ADA Remediation

*University of Florida — Training & Organizational Development | Internal Use Only*

---

## Overview

This internal document was designed by the Training & Organizational Development Instructional Design Team to provide structure and guidance for making Articulate Rise courses ADA compliant with WCAG 2.2 AA standards.

Instructional designers will review and update each Rise course per the guidelines below, re-upload the course into myTraining using Replace Content, and mark the course as "ADA 2.0 Complete" on the OLT Numbers Report in Teams.

---

## ADA Process

- Use **Poppins at a minimum of 12pt** font, or another sans-serif font.
- Add scene descriptions to any scenario blocks (see [Scene Descriptions](#scene-descriptions) below).
- Add alternative text to images (see [Alt Text](#alt-text) below).
- Correct hyperlinks from full web addresses to descriptive titles/text.
- Check and replace broken links (link to the HR toolkit webpage, not the document link, whenever possible).
- Check text contrast on blocks where an image has overlaid text. Adjust the image overlay opacity to darker if necessary (80%).
- Add closed captioning to any videos (see [Closed Captions](#closed-captions) below).
- For custom blocks, include focus order and alt text (see [Custom Blocks](#custom-blocks) below).
- Remove any drag-and-drop interactions and replace with another interaction or text (see [Remove Drag and Drop](#remove-drag-and-drop-interactions) below).
- Use H2 style on the overview page (Title, Length, Audience, etc.). If other text blocks have custom headings (H3, H4, H5), add those as well via the dropdown under the paragraph symbol/option.
- Make sure tables have a header and a caption.
- Remove screen recording videos, as they require audio descriptions.
- Make any Storyline blocks ADA-compliant before importing into Rise. Store the .story file in the appropriate folder for the Rise course in Teams and/or the U drive.
- Re-upload the Rise course in myTraining using [Replace Content](https://uflorida.sharepoint.com/:b:/r/teams/HR_TrainingOrganizationalDevelopment/Shared%20Documents/myTraining%20System/myTraining_Partner_Program/Toolkit_Instruction_Guides/28Aug2024_myTraining_ReplaceContent_Guide.pdf?csf=1&web=1&e=NFhJcB).
- Update the [OLT Numbers spreadsheet](https://uflorida-my.sharepoint.com/:x:/g/personal/david_smith2_ufl_edu/IQAJbVPu6X__S4pl0jChtT8-AdlhgztRoXZBtg1pqhdS68E?e=TrSV3m), column L, to "ADA2.0 Complete."

> **Note:** Use text blocks to convey important information rather than importing images of text (such as a PowerPoint slide or complex graphic), which cannot be read by assistive tools.

## Scene Descriptions

- Use the scene description field for the scene background and character. You do not need to include the character dialogue here — that will be read by the screen reader.
- Navigate to **Edit Content** in the upper left corner to access scene descriptions.
- Add your alt text under each scene description. Describe the setting and the body language of the character.

---

## Alt Text

- Add alternative text to images to improve accessibility for learners using screen readers. After adding an image in Rise 360 (or your cover page in Reach 360), click the **Edit** link for the image and choose **Edit alt tag** from the menu.
- Enter descriptive text in the window that appears and click **OK**.
- Images are marked decorative by default. Leave the field blank to keep it null — screen readers will skip it. (Cover photos are ignored by screen readers automatically.)
- Alt text should be **1–2 sentences, 20–30 words maximum.**
- Once alt text is added, an **ALT tag label** displays in the corner of the image in the editor (not in published output) as a visual indicator. Click the ALT tag to edit the alt text.

## Closed Captions

### Preparing Your Caption Files

Rise 360 supports **VTT files** generated by third-party captioning services such as YouTube, Amara, and 3PlayMedia.

> **Do not** use the Teams browser shortcut to generate captions — put the video in a Teams file, open in browser, and generate from settings. This method should not be used.

**Formatting tips:**

- Use the timestamp format: `hours:minutes:seconds.milliseconds` with two digits each for hours, minutes, and seconds, and three digits for milliseconds. Example: `00:00:05.001 → 00:00:08.200`
- Rise 360 supports bold, italic, and underline formatting in caption files, as well as voice tags to indicate which person is speaking. All other formatting tags (font, font color) will be ignored.
- Captions wrap automatically if too long. To force a break at a specific point, add one line break in the caption file.
- To split a caption into two simultaneous caption boxes (e.g., two speakers), add two line breaks in the caption file.
- If letters or characters are unexpectedly replaced by symbols, ensure your caption file is encoded in **UTF-8**.
- Closed captions will only display when the course is hosted on a web server or LMS — not when viewed from a local hard drive.

### Adding Closed Captioning to a Video

1. Mouse over a video block or interactive block with a video, then click the **Edit** button in the upper left corner of the block.
2. Click **Edit** next to the video and select **Manage captions**.
3. Enter the language in the **Add a new cc file** field and select it from the list.
4. Click **Upload CC File** and select the corresponding .vtt file.
5. Repeat the previous two steps to add additional languages (no limit).
6. To remove an uploaded file, hover over the language in the list and click the trash icon.

Learners can then select a caption language from the **CC** button during playback.

### Downloading and Deleting Caption Files

1. Mouse over the video block and click the **Edit** button in the upper left corner.
2. Click **Edit** next to the video and select **Manage captions**.
3. Hover over a language in the captions list to see the download and delete icons.
4. Click the **Download** icon to download the .vtt file. If it opens in a new window instead of downloading, right-click the icon and select **Save Link As**.
5. Click the **trash can icon** to delete the associated caption file.

## Custom Blocks

For custom blocks, adjust focus order and add alt text.

1. Select **Focus Order**.
2. Drag objects into the correct order for a screen reader to read.
3. For any object that contains text or an image conveying meaning or context, include alt text.

---

## Remove Drag and Drop Interactions

Remove any drag-and-drop interactions (such as sorting activities) and replace them with an alternative interaction or a text block.

---

## Headers

Add header styles on the overview page if not already set. Use the paragraph style dropdown to apply H2 for the overview page headings (Title, Length, Audience, etc.), and H3/H4/H5 for any further nested headings in other text blocks.

> **Note:** Header styles should be added to blocks for all future Rise courses — see the publishing guide for more details.
