# Word Doc WCAG 2.2 AA Checklist

*University of Florida — Training & Organizational Development | Human Resources*

---

## Document Structure

- [ ] Heading 1 for the title; Heading 2/3 for sections; no skipped levels
- [ ] Body text uses Normal; Strong only for brief emphasis; avoid manual formatting
- [ ] Steps are numbered; guidance uses bullets with the Bulleted List style (not hyphens or asterisks)
- [ ] Notes and other callout paragraphs use the Note style, with the first word set to Strong
- [ ] Actions to be taken on screen are set to Clicks (e.g., in "click OK," "OK" is set to Clicks)
- [ ] Automatic Table of Contents inserted and updated (required for docs > 3 pages)

---

## Text and Color

- [ ] Sufficient color contrast (at least 4.5:1 for normal text — use Accessibility Checker)
- [ ] Color alone is not used to convey meaning (use text labels or patterns as well)
- [ ] Avoid images of text; if unavoidable, provide equivalent text nearby

---

## Links

- [ ] Descriptive link text that states the destination or action (not "Click here" or a raw URL)
- [ ] Links are visually distinguishable (e.g., underline or default hyperlink style)

---

## Images and Media

- [ ] Alt text added for all meaningful images, charts, and tables
- [ ] Decorative images are marked as decorative
- [ ] Alt text is concise and contextual (explains why the image matters)
- [ ] For charts/complex graphics: brief alt text provided plus a nearby data table or longer text alternative
- [ ] All images set to **In Line with Text** to preserve reading order

---

## Tables

- [ ] Table Design options enabled: Header Row, Banded Rows, First Column
- [ ] Header rows repeat across pages; rows do not break across pages
- [ ] No merged or split cells; structure is simple
- [ ] Caption added (if helpful) and alt text describes the table's purpose
- [ ] No bullets or lists within table cells
- [ ] No images within table cells
- [ ] Tables are used for data only, not for layout or formatting (avoid unless absolutely necessary)

---

## Navigation and Input

- [ ] Logical reading order maintained (no floating text boxes that disrupt flow)

---

## Metadata and Framing

- [ ] Title set in **File > Info > Properties > Title** (matches Heading 1 and file name)
- [ ] Standard T&OD header and footer applied
- [ ] Footer includes: Department | Unit and the guide title

---

## Final Checks

- [ ] Word's Accessibility Checker has been run and all flagged issues resolved
- [ ] Spot-check headings, links, images, tables, and TOC after any edits
- [ ] Document is set to open read-only by default when publishing (department standard)
